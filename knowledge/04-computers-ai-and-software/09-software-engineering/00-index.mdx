# 👨‍💻 Software Engineering

## 📚 Overview

Software engineering is the discipline of building software systems reliably, efficiently, and at scale. This shelf covers the full spectrum: from Robert Martin's Clean Code to Steve McConnell's Code Complete, from the Pragmatic Programmer to managing technical debt, from RESTful API design to GraphQL.

These books are the foundation of professional software development — the knowledge that separates hobbyists from engineers.

## 🔗 Category Boundaries

- **Software Design Principles** covers architectural patterns and design principles.
- **Code Quality & Craft** covers code-level practices.
- **DevOps & Platform Engineering** covers deployment and operations.

This category focuses on the **engineering discipline** — the practices, processes, and skills of professional software development.

## 📖 Topics Covered

- Clean code: writing code that's easy to read and maintain
- Code complete: the craft of software construction
- API design: REST, GraphQL, gRPC, webhooks
- Technical leadership and staff engineering tracks
- Managing technical debt
- Engineering productivity and developer experience
- Remote engineering teams

## ⭐ Essential Books

- **Clean Code** — Robert C. Martin: A handbook of agile software craftsmanship — how to write code that's easy to read, change, and maintain.
- **Code Complete** — Steve McConnell: The definitive practical handbook of software construction.
- **The Pragmatic Programmer** — Andrew Hunt & David Thomas: A journeyman's guide to the craft of software development.
- **The Art of Readable Code** — Boswell & Foucher: Simple and practical techniques for writing code that's easy to understand.
- **Technical Leadership and Staff Engineering** — The path from senior engineer to technical leader.

## 🚀 Why This Category Matters

Software engineering is the discipline that turns code into products. These books teach you how to build software that's not just functional, but reliable, maintainable, and a joy to work with.
