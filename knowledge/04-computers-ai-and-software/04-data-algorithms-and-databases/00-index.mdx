# 🗄️ Data, Algorithms & Databases

## 📚 Overview

Data structures, algorithms, and databases are the three pillars of practical computer science. This shelf covers all three: from Cormen's Introduction to Algorithms to Knuth's Art of Computer Programming, from database internals to SQL theory, from Grokking Algorithms to the Algorithm Design Manual.

Whether you're preparing for technical interviews or building production systems, these books provide the foundational knowledge you need.

## 🔗 Category Boundaries

- **Theory of Computation** covers computability and complexity theory.
- **System Design & Architecture** covers the architecture of data-intensive systems.
- **Artificial Intelligence** covers ML-specific algorithms.

This category focuses on the **practical tools** — data structures, algorithms, and databases — that every software engineer needs.

## 📖 Topics Covered

- Advanced data structures: trees, graphs, heaps, hash tables
- Algorithm design paradigms: divide-and-conquer, dynamic programming, greedy
- Classical algorithms: sorting, searching, graph algorithms
- Database internals: storage, indexing, query optimization
- Relational model and SQL theory
- NoSQL data models: document, key-value, column-family, graph
- Distributed database theory and the CAP theorem

## ⭐ Essential Books

- **Introduction to Algorithms** — Cormen, Leiserson, Rivest, Stein: The definitive algorithms textbook, known as "CLRS."
- **The Art of Computer Programming, Vol. 1** — Donald Knuth: The bible of algorithmic analysis by the father of the field.
- **The Algorithm Design Manual** — Steven Skiena: A practical guide to algorithm design with real-world examples.
- **Grokking Algorithms** — Aditya Bhargava: An illustrated, accessible introduction to algorithms.
- **Database Internals** — How databases work under the hood: storage, indexing, and query processing.

## 🚀 Why This Category Matters

Data structures, algorithms, and databases are the tools that every software engineer uses every day. Mastering them is the difference between writing code that works and writing code that scales.
